**Course Project | Debre Berhan University**  
**Developers:** Fantayenesh Worku
---

## Project Overview

This project builds a machine learning pipeline to predict whether a blood donor
will return to donate blood in the next campaign, using the RFMTC
(Recency, Frequency, Monetary, Time, Cholesterol) dataset.

A **Random Forest Classifier** (`n_estimators=100`) is trained on a preprocessed
version of the dataset after handling missing values using **KNN Imputation** (`k=5`)
and performing a stratified **80/20 train/test split**.

## Dataset

The RFMTC blood donation dataset contains records of blood donors with the following features:

| Column | Description |
|---|---|
| Recency | Months since the last donation |
| Frequency | Total number of donations |
| Monetary | Total volume of blood donated (cc) |
| Time | Months since the first donation |
| Donor Age | Age of the donor |
| Blood Type | ABO blood group |
| Location | Donation location (Hospital, Clinic, Mobile Bus) |
| **Target** | **1 = will donate, 0 = won't donate** |

## Methodology

### Missing Value Handling
Missing values are imputed using **K-Nearest Neighbors (KNN) Imputation** with
`n_neighbors=5`. This method estimates each missing value from the `k` most similar
records (based on the other non-missing features), preserving the local data distribution
better than simple mean/median imputation.

### Feature Engineering
Three derived features are computed from the RFMTC columns:
- `donation_density` = Frequency / Time — donation rate per month
- `recency_score` = 1 / (Recency + 1) — fresher donors score higher  
- `high_frequency_flag` = 1 if Frequency > median(Frequency)

### Model
**Random Forest Classifier**
- `n_estimators = 100`
- `random_state = 42`
- `class_weight = 'balanced'`
- Train/test split: **80% / 20%** (stratified)
- Cross-validation: **5-fold**
